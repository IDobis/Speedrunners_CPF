Sobre o projeto

Esta aplicação usa o universo de speedruns para organizar uma comunidade de jogadores que buscam concluir jogos, fases ou desafios no menor tempo possível.

SR

Base temática: speedrun

Speedrun é uma modalidade dos games em que o objetivo é terminar um jogo inteiro, uma fase ou uma categoria específica no menor tempo possível. O projeto usa essa ideia para simular uma pequena comunidade de competidores, chamados speedrunners, que precisam ser cadastrados e consultados pela organização.

CT

Categorias, técnicas e validação

No speedrun, cada tentativa costuma ser chamada de run e pode seguir regras próprias, como categorias que permitem ou proíbem glitches. A comunidade também valoriza provas, transmissões e temporizadores para confirmar os tempos. Por isso, a aplicação foi pensada como um painel de controle com dados confiáveis, lista organizada e consulta externa.

HT

HTML semântico

A estrutura usa tags com significado: header para o cabeçalho, nav para o painel de navegação, main para o conteúdo principal, section para cada tela, form para entradas de dados, table para a lista, caption para descrever a tabela e article para blocos independentes de informação. Isso melhora leitura do código, acessibilidade e compreensão pelo navegador.

BX

CSS, Box Model e hierarquia visual

O CSS controla margem, preenchimento, borda, largura e altura dos elementos, que são partes do Box Model. As telas usam cartões, botões, campos e tabelas com espaçamento consistente para separar grupos de conteúdo. A hierarquia visual aparece nos títulos maiores, textos de apoio menores, cores de destaque e mensagens de erro ou sucesso.

FG

Flexbox, Grid e responsividade

Flexbox foi usado no cabeçalho, nos botões de ação e nos formulários que precisam alinhar itens lado a lado. Grid foi usado nas grades de campos, resumo, endereço e conceitos, porque facilita organizar colunas iguais. As Media Queries mudam essas colunas para uma coluna única em telas menores, mantendo o layout fluido em celulares e computadores.

JS

JavaScript e eventos

O JavaScript controla as interações: cliques nos botões de navegação, envio do formulário, digitação nos campos, saída do campo de CEP e ações de editar ou excluir. As funções em português deixam claro o comportamento esperado, como mostrarTela, validarFormularioUsuario, salvarUsuario, consultarCep e alternarTema.

VF

Validação em tempo real

Nome, e-mail, CPF e idade são validados enquanto o usuário digita. O CPF recebe máscara, passa por cálculo dos dígitos verificadores e também é comparado com os CPFs já salvos para evitar cadastro repetido. O atributo aria-invalid e as mensagens em aria-live ajudam a comunicar erros de forma acessível.

LS

LocalStorage

O LocalStorage guarda dados diretamente no navegador. A lista de usuários é convertida para texto com JSON.stringify antes de salvar e volta a ser array com JSON.parse ao carregar a página. Assim, os registros continuam disponíveis depois de atualizar ou fechar o site.

API

Fetch API e ViaCEP

A tela de CEP usa fetch para consultar a API pública ViaCEP. Quando o CEP tem oito números, o JavaScript envia a requisição, espera a resposta em JSON e preenche rua, bairro, cidade e estado. Se a API retornar erro ou o CEP não existir, a aplicação limpa os campos e mostra uma mensagem.

TE

Tema claro e escuro

O botão de tema altera um atributo no elemento html. O CSS lê esse atributo e troca variáveis de cor, como fundo, texto, borda e destaque. A escolha também é salva no LocalStorage para que o tema preferido seja restaurado quando a página for aberta novamente.

AC

Acessibilidade, imagens locais e ícones

A interface usa botões reais, labels ligados aos campos, mensagens com role status e aria-live, foco visível e textos claros. As imagens principais foram trocadas por arquivos locais da pasta Img, evitando dependência de serviços externos. Os botões e cards receberam ícones por CSS, usando pseudo-elementos para reforçar a função de cada ação.

UI

Estilização final e temas visuais

A camada final de CSS adicionou transições, sombras suaves, estados de hover, foco acessível e ícones nos controles principais. O tema claro usa uma leitura inspirada no Google, com fonte Inter, superfície branca, azul de ação e apoio em vermelho, amarelo e verde. O tema escuro usa uma leitura inspirada na Apple, com pilhas de fonte SF Pro, fundo preto, painéis translúcidos, bordas mais claras e azul de destaque. Também foram adicionados ajustes responsivos para tablets e celulares da Apple, incluindo larguras próximas de iPad, iPhone, áreas seguras do sistema e botões maiores para toque.

RF

Referência usada

A explicação temática foi baseada no artigo O que é speedrun?, do Tecnoblog, que apresenta a modalidade, a ideia de terminar jogos rapidamente, o papel da comunidade, das transmissões e dos temporizadores.
